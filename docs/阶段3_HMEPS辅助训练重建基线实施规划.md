# 阶段 3：HMEPS 辅助训练重建基线实施规划

## 目标体验

针对空预测集中于少数困难 LPW 受试者的问题，引入已登记为 CC BY 4.0 的 HMEPS 人眼训练样本扩大外观与采集条件覆盖，重新建立独立 64×64 FP32 与 INT8 基线。

## 覆盖审计依据

- 标准 PTQ 验证集 32 个空预测全部来自 LPW 受试者 03。
- 标准 PTQ 测试集 11 个空预测全部来自 LPW 受试者 11。
- LPW 训练集 16 名受试者各约 139—150 张，数量分配均衡；问题不是单一训练受试者样本数失衡。
- HMEPS 有 3,527 张有效非空人眼训练样本，许可证为 CC BY 4.0，项目总控已规定其只可作为训练辅助数据。

## 数据边界

- 训练：2,371 张 LPW + 3,527 张 HMEPS，共 5,898 张。
- 验证与测试：继续使用原固定 LPW 受试者清单，不加入 HMEPS。
- 不改变 LPW 切分，不把验证或测试困难样本回流训练。
- 新实验使用独立编号，不覆盖 LPW-only 基线。

## 输入输出

1. FP32 训练输入：`train.csv` 混合训练清单和独立配置。
2. FP32 输出：最佳检查点、验证复核、一次测试指标及逐样本结果。
3. INT8 输入：混合训练最佳 FP32 检查点；校准仍只使用目标域 LPW 训练样本。
4. INT8 输出：ESP32-S3 `.espdl`、验证/测试同口径指标与相对 LPW-only 基线变化。

## 失败路径

- 混合训练验证指标或空预测不改善：保留负结果，不修改测试集和切分。
- HMEPS 域差异导致整体退化：停止辅助数据路线，标准部署仍使用 LPW-only 基线。
- INT8 重新出现几何退化：不以边界收益掩盖中心误差和空预测率代价。

## 验收方式

- 验证集独立复核必须与训练期最佳指标一致。
- 测试集在检查点固定后只评价一次。
- 报告 FP32 与 INT8 的 Dice、Boundary IoU、中心误差和空预测率。
- 全仓库 Ruff 和自动测试通过。
