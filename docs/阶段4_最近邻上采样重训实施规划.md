# 阶段 4：最近邻上采样重训实施规划

## 目标体验

建立一个明确区分于原双线性模型的 ESP32-S3 部署候选：4 个解码器上采样统一使用最近邻插值，重新完成 FP32 训练、独立评价、INT8 量化与固定测试，确认精度后再进行固定图上板验证。

## 固定边界与成功标准

- 只改变 `LightUNet` 的 4 个上采样插值模式；通道数、卷积、损失、训练集、受试者切分、随机种子和输入尺寸保持不变。
- 新模型使用独立配置、运行目录和精度报告，不继承原双线性模型的 FP32 或 INT8 结论。
- PTQ 参数与统一阈值只在验证集选择，测试集只评价锁定后的唯一候选。
- ONNX 图必须包含 4 个 `mode=nearest` 的 Resize，不得包含双线性 Resize。
- ONNX 导出时由模型代码为最近邻 Resize 显式提供全范围 ROI，避免空可选输入；该适配不得改变 PyTorch 与 ONNX 数值，必须通过自动对齐测试和板端加载共同验收。
- `.espdl` 必须由 ESP-DL 3.3.1 在 ESP32-S3 上成功加载；固定输入推理不得在 Resize 参数解析处崩溃。
- 若新 INT8 精度未满足原有中心误差与空预测率硬约束，停止上板并保留失败结果，不隐瞒架构代价。

## 改动范围与职责

- 模型层：`models/common.py` 仅把 `UpSample` 的插值模式由双线性改为最近邻，保留后续 1×1 卷积、BN 与 ReLU。
- 测试层：补充上采样模式与 ONNX Resize 属性检查，防止再次导出双线性算子。
- 训练层：复用 `training/train_fp32_baseline.py`，通过独立配置和输出目录重新训练。
- 评价层：复用 `evaluation/evaluate_fp32_checkpoint.py`，分别评价固定验证集和测试集。
- 量化层：复用 ESP-PPQ 搜索流程；以新 FP32 检查点为唯一输入，生成独立 `.espdl`、ONNX 和指标。
- 部署层：仅在软件精度与图结构验收通过后替换板端模型分区内容；固件推理服务不改变业务职责。

## 输入与输出

- 输入：LPW+HMEPS 固定训练清单、LPW 固定验证/测试清单、最近邻 LightUNet、ESP-PPQ 1.3.6。
- 输出：新 FP32 检查点、验证/测试指标、PTQ 搜索与锁定结果、最近邻 ONNX、`.espdl`、板端加载与固定图推理日志。

## 失败路径

- 训练或评价失败：保留独立运行目录和错误，不覆盖旧模型产物。
- ONNX 仍出现非最近邻 Resize：停止量化与上板，修正模型导出链路。
- Resize 仍含空 ROI 输入，或 ONNX 与 PyTorch 最近邻输出不一致：停止上板，保留导出失败证据。
- 没有 PTQ 候选满足几何硬约束：停止上板，单独评审是否接受精度变化。
- ESP-DL 仍在加载阶段崩溃：核对导出图与算子日志，不回退为直接修改 ONNX。
- 板卡或串口不可用：完成软件闭环并记录硬件阻塞，不重复烧录。

## 验收方式与建议人工操作

1. 运行模型单元测试和 ONNX 属性检查。
2. 完成 FP32 重训，并独立复核验证集与测试集指标。
3. 完成 PTQ 验证搜索，锁定唯一配置后执行一次测试集评价。
4. 检查 `.espdl` 与 ONNX 产物哈希、Resize 数量和模式。
5. 保持开发板连接 `COM5`；烧录后监视模型加载、输出形状、固定图摘要、内存和连续推理状态，验收结束按 `Ctrl+]` 退出监视器。
