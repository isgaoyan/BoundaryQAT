# BoundaryQAT 文献与设计评估

## 结论

BoundaryQAT 适合作为边缘 AI 与机器视觉求职项目，核心价值是“瞳孔分割、INT8 量化和 ESP32-S3 部署的工程闭环”。它不应以未经验证的算法首创为目标。

## 关键文献与设计影响

| 文献/资料 | 对项目的影响 |
|---|---|
| [RITnet](https://arxiv.org/abs/1910.00694) | 轻量眼部分割已有成熟基线；小于 1 MB 的模型不是创新点，因此本项目重点放在 MCU 可部署性、量化误差和实测性能。 |
| [OpenEDS](https://arxiv.org/abs/1905.03702) | 真实近眼图像包含多名受试者与像素级标注；数据集切分必须按受试者或序列隔离，避免相邻帧泄漏。 |
| [RIT-Eyes](https://arxiv.org/abs/2006.03642) | 合成眼图可缓解标注成本，但需要评估向真实眼图的泛化；合成数据不能作为唯一测试来源。 |
| [Boundary IoU](https://openaccess.thecvf.com/content/CVPR2021/html/Cheng_Boundary_IoU_Improving_Object-Centric_Image_Segmentation_Evaluation_CVPR_2021_paper.html) | 普通 IoU/Dice 对边界误差不够敏感；采用 Boundary IoU 作为主边界指标，并预先固定边界带宽。 |
| [整数语义分割量化](https://arxiv.org/abs/2011.14504) | 分割模型的全整数部署与量化误差已有研究；项目应比较硬件一致的 PTQ、QAT/TQT，而非重复提出通用量化框架。 |
| [ESP-DL 量化说明](https://docs.espressif.com/projects/esp-dl/en/latest/tutorials/how_to_quantize_model.html) | ESP32-S3 使用对称、二的幂、逐张量量化。所有离线实验都必须服从该约束，避免 PC 端收益无法上板复现。 |
| [Boundary-Aware Quantization 预印本](https://arxiv.org/abs/2607.01478) | 已有工作研究量化与决策边界的关系；本项目不得宣称“首次测量边界量化误差”。该文与像素级瞳孔边界不同，可作为问题背景，不作为方法直接对比。 |

## 已排除的原方案

原设想按“边界敏感通道”单独调整激活量化 scale。该方案依赖逐通道激活量化，而 ESP32-S3 的目标后端采用逐张量策略，不能作为可部署实现的核心。

同时，在常见量化定义中，增大 scale 会增大量化步长，而不是让量化更精细；缩小 scale 则可能导致截断。因此不能直接使用“scale 乘以大于 1 的因子以保护边界”的表述。

## 推荐实验最小集

1. **FP32 LightUNet**：建立浮点精度、Dice、Boundary IoU 与中心 MAE 基线。
2. **PTQ-INT8**：使用 ESP-PPQ 默认 ESP32-S3 配置，建立可部署量化基线。
3. **QAT/TQT-INT8**：仅在可导出 ESP-DL 模型的前提下实施，比较训练期量化适配收益。
4. **Boundary-guided tuning**：在验证集上，以“Dice 损失不超过预设阈值时最大化 Boundary IoU”为固定规则选择校准、阈值或 TQT 配置；最终测试集只报告一次。
5. **ABeDice + QAT/TQT（可选）**：仅在前三组完成且时间允许时加入，作为边界损失对照。

## 结论表达边界

- 可以说：量化对瞳孔边界和中心定位的影响在此任务、此数据和此硬件约束下被系统测量。
- 不可以说：首次发现边界量化退化、提出通用最优量化算法，或具备临床诊断能力。
- 只有在完成真实板端测量后，才能说“已部署”或“实时”；只有测得能耗后，才能报告 mJ/帧。

## 对求职呈现的含义

最有价值的成果不是单一精度数字，而是同一模型在 FP32、INT8 模拟和 ESP32-S3 实机上的一致证据，以及你对数据域差异、指标取舍、部署失败和模型缩减的解释。
