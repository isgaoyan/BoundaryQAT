# 阶段 3：边界指标驱动阈值优化实施规划

## 目标体验

在不修改 LightUNet 权重、ESP32-S3 INT8 图结构和运行路径的前提下，使用验证集选择统一的二值化阈值，验证能否在整体分割与中心定位损失受控时改善瞳孔边界质量。

## 方法命名与边界

- 本阶段命名为 `Boundary-guided tuning`，不命名为 QAT。
- 固定阶段 2 的 64×64 FP32 检查点、128 张校准清单、ESP32-S3 逐张量 INT8 配置和最大连通域后处理。
- 只搜索全数据统一的概率阈值，不增加输入相关分支、动态 scale 或额外模型算子。
- 验证集用于选择；测试集在阈值固定后只评价一次。

## 候选与选择标准

- 候选阈值：0.30 至 0.70，步长 0.01；标准基线为 0.50。
- 合格条件：验证 Dice 不低于标准 PTQ 0.005，中心误差不高于标准 PTQ 0.5 px，空预测率不高于标准 PTQ 1 个百分点。
- 合格候选中优先选择 Boundary IoU 最高者；并列时依次选择 Dice 更高、中心误差更低、与 0.50 更接近者。
- 若没有新候选优于标准阈值，则保留 0.50 并记录负结果。

## 模块职责与输入输出

1. 量化服务：复用阶段 2 配置和固定校准清单，生成量化图与候选 `.espdl`。
2. 预测缓存：对固定验证集或测试集执行一次量化推理，缓存概率图和真值，不在阈值循环中重复推理。
3. 选择器：对验证集候选阈值计算全部主指标和诊断指标，按预设约束选择最终阈值。
4. 最终评价：只读取已固化的选择文件，对测试集同时报告标准阈值与最终阈值。
5. 图结构核对：比较阶段 2 与阶段 3 导出 ONNX 的节点类型、连接顺序及张量形状，确认阈值优化不改变模型图。

## 失败路径

- 若所有新阈值均无受约束边界收益，保留标准阈值 0.50，不扩大搜索空间或读取测试集调参。
- 若量化导出或推理失败，保留中间产物和错误，不改变数据切分。
- 若测试集收益与验证集方向不一致，如实报告泛化失败，不二次选择阈值。

## 验收方式

- 保存全部候选阈值及验证指标、选择约束、最终阈值和一次性测试结果。
- 全量自动测试与 Ruff 检查通过。
- 阶段 2 与阶段 3 导出模型图结构一致。
- 明确报告边界收益、Dice、中心误差与空预测率代价。

## 建议人工操作

- 人工检查标准阈值与最终阈值的最差案例，重点关注小瞳孔、强反光和空预测样本。
- 开发板到达后，将最终阈值作为统一后处理常量，与候选 `.espdl` 一同做固定图板端验证。
