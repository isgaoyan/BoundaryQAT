# 阶段 1：具体数据流程

## 阶段边界

阶段 1 全部运行在电脑端，目标是得到可信的 FP32 模型和评估结果，**不会把图像送入 ESP32-S3**。阶段 1 的最终交付物中，只有最佳 FP32 检查点会在阶段 2 进入 ESP-PPQ，量化并导出为 `.espdl`；真正的相机图像进入 ESP32-S3 属于阶段 4。

当前已完成统一数据预处理、固定受试者切分、数据加载器、FP32 训练器、固定后处理、指标计算与独立测试评估。主结果见[阶段 1：FP32 基线结果](阶段1_FP32基线结果.md)。

现有 `LightUNet` 原型的注释和主要测试仍以 `64×64` 为默认输入，而阶段 1 数据产物是 `128×128`。该网络结构理论上可接收 128×128，但正式训练前仍须增加 128×128 端到端测试，并把训练配置中的输入尺寸固定下来；不能只根据结构推断视为已经验收。

## 一、训练数据链路

| 步骤 | 进入的数据 | 接收部件 | 部件行为 | 输出及去向 | 最终作用 |
|---|---|---|---|---|---|
| 1. 来源筛选 | LPW 图像/掩码；HMEPS 原图、掩码和 `annotations.csv` | 数据准备程序 `prepare_pupil_segmentation_data.py` | LPW 按同名文件配对；HMEPS 只保留 `exp=h`、`eye=1`、`blink=0` 的人眼非眨眼样本 | 6,798 对合法图像与掩码进入统一预处理 | 排除错配、鼠眼、无眼和眨眼数据 |
| 2. 格式统一 | 原始彩色或灰度眼图、来源不同的掩码 | 图像/掩码读取部件 | 原图转 8-bit 单通道；LPW 掩码直接二值化；HMEPS 掩码只读取红通道后二值化 | 单通道眼图和取值为 `0/255` 的瞳孔掩码进入几何变换 | 让不同来源具备统一输入语义 |
| 3. 几何变换 | 尺寸不一致的图像与掩码 | 等比例缩放与补边部件 | 图像用双线性插值、掩码用最近邻插值；二者使用相同比例缩放并居中补边至 `128×128` | `images/`、`masks/` 中的统一 PNG；变换参数写入 `samples.csv` | 保持瞳孔椭圆形状，避免直接拉伸造成标注失真 |
| 4. 数据切分 | `samples.csv` 中的来源、受试者、序列和路径 | `create_stage1_splits.py` 与 `stage1_baseline.json` | LPW 按固定名单切成 16/3/3 名受试者；HMEPS 只加入训练集；空掩码写入排除清单 | `train.csv`、`validation.csv`、`test.csv`、`excluded.csv` 与 `summary.json` 进入数据加载器和审计流程 | 防止同一受试者泄漏到不同集合，并使排除行为可追溯 |
| 5. 批次加载 | 清单指向的 `128×128` 灰度 PNG 与二值掩码 PNG | `PupilSegmentationDataset` 与 PyTorch DataLoader | 读取并验证图像和掩码；图像除以 255；掩码转换为 `0/1`；组成固定批次 | 图像张量 `[B,1,128,128]` 与真值张量 `[B,1,128,128]` 进入 LightUNet | 给模型提供可训练的标准批次 |
| 6. 前向与学习 | 图像批次、真值掩码、当前模型参数 | FP32 LightUNet 与 `train_fp32_baseline.py` | LightUNet 输出每像素瞳孔概率；BCE+Dice 比较概率图和真值；Adam 反向传播并更新权重 | 更新后的 FP32 权重继续下一批训练；每轮验证结果进入检查点选择器 | 学会从眼图中分出瞳孔区域 |
| 7. 模型选择 | 每轮模型权重和验证集预测 | 训练器的验证与检查点部件 | 只在验证集计算固定指标，以 Dice 最高、Boundary IoU 次优选择最佳轮次，并执行早停 | 第 31 轮最佳 FP32 检查点进入独立评估；测试集不参与选择 | 避免用测试集调参 |
| 8. 最终评估 | 最佳检查点、固定 LPW 测试集 | `evaluate_fp32_checkpoint.py` | 生成最大连通域掩码、中心坐标、指标 JSON、逐样本 CSV 和三联预览 | 测试 Dice 0.9021、Boundary IoU 0.6474、中心误差 1.467 px；检查点交给阶段 2 | 建立后续 INT8 量化的精度上界 |

## 二、单张图像在 FP32 基线中的数据流

```text
128×128 灰度眼图
  → 数据加载器：转为 [1, 1, 128, 128] 的 FP32 张量并归一化
  → LightUNet 编码器：提取由浅到深的眼部纹理与区域特征
  → LightUNet 瓶颈层：整合瞳孔与周围组织的上下文
  → LightUNet 解码器 + 跳跃连接：恢复空间分辨率并补回边缘细节
  → 输出层：生成 [1, 1, 128, 128] 的瞳孔概率图
  → 阈值部件：逐像素比较固定阈值，生成 0/1 二值掩码
  → 中心计算部件：对预测瞳孔前景计算中心坐标 (x, y)
  → 指标部件：与真值掩码/真值中心比较
  → 输出 Dice、Boundary IoU、中心 MAE 和预测叠加图
```

各部件的责任必须分开：LightUNet 只产生概率图；阈值与中心计算属于后处理；指标部件只负责评估，不参与最终测试集上的模型选择。

## 三、阶段 1 与 ESP32-S3 的交接

阶段 1 不直接输出 ESP32-S3 可运行模型，交接路径如下：

```text
阶段 1 最佳 FP32 检查点
  → 阶段 2 ESP-PPQ：使用校准集把 FP32 权重/激活映射为 ESP32-S3 支持的逐张量 INT8
  → ESP-PPQ 导出 `.espdl`
  → 阶段 4 ESP32-S3 的 ESP-DL：加载 `.espdl` 并执行推理
```

阶段 4 的预期图像路径是：OV2640 输出相机像素 → ESP32-S3 相机外设与 DMA 写入帧缓冲区 → CPU 进行灰度化、等比例缩放、补边和 INT8 输入变换 → ESP-DL 执行模型 → 输出概率/分割张量写入内存 → CPU 阈值化并计算中心 → 通过串口、屏幕或上层控制模块输出瞳孔中心。该链路必须等实机接线和部署代码确定后再锁定具体引脚、缓冲区位置和输出接口。

## 四、数据契约

| 交界面 | 必须固定的内容 |
|---|---|
| 预处理 → 数据加载器 | `128×128`、8-bit 单通道；掩码只含 `0/255`；清单可追溯到原始文件 |
| 数据加载器 → LightUNet | 图像 `[B,1,128,128]` FP32；掩码 `[B,1,128,128]` 且只含 `0/1` |
| LightUNet → 后处理 | 每像素瞳孔概率图；阈值必须写入配置，不得在测试集上调整 |
| 后处理 → 指标 | 二值掩码、预测中心、真值掩码、真值中心使用同一坐标系 |
| 阶段 1 → 阶段 2 | 最佳检查点、模型结构、输入尺寸、归一化参数、固定校准清单和 FP32 测试结果 |

## 五、失败路径与验收

- 发现图像/掩码错配、尺寸不一致或非法掩码值：数据准备立即失败，不允许静默跳过。
- 21 个空掩码样本：训练前必须在配置中明确“作为负样本保留”或“排除”，并记录数量；不能临时人工删除。
- 64×64 定位不稳定：阶段 1 先保留 `128×128` 基线；后续从原始数据重新生成 64×64，不能二次缩放 128×128 产物。
- 当前模型原型以 64×64 为默认说明：开训前必须用 128×128 的真实数据批次验证输入、输出、显存占用和训练稳定性，再确认阶段 1 的正式尺寸契约。
- 验收通过条件：固定切分可复现；数据与指标单元测试通过；最佳检查点可重复得到；测试集同时产生 Dice、Boundary IoU、中心 MAE、指标 JSON 和预测可视化。
