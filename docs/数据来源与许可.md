# 数据来源与许可

本文登记阶段 1 的实际数据来源、目录约定与使用边界。下载数据不进入 Git；切分清单、配置和生成的评估结果可进入版本控制。

## LPW 原始数据

- 名称：Labeled Pupils in the Wild（LPW）。
- 官方页：[Collaborative AI — LPW](https://collaborative-ai.org/research/datasets/LPW/)。
- 本地位置：`data/LPW/<受试者编号>/<视频编号>.avi`；同名 `.txt` 每行对应视频的一帧，记录瞳孔中心 `x y`。
- 规模：22 名受试者、66 段视频、约 130,856 帧；原始下载包约 2.4 GB。
- 使用限制：官方页面要求仅用于非商业科研用途，并要求引用 Tonsen et al., ETRA 2016。原始压缩包 README 提及 CC BY-NC 3.0；本项目按官方页面更明确的“非商业科研用途”作为使用边界。

## LPW GT 分割掩码

- 下载页：[Kaggle — LPW GT Segmentation Maps](https://www.kaggle.com/datasets/cuneytakinlar/lpw-gt-segmentation-maps)。
- 关联说明：[pupil-ellipse-fit-error 项目 README](https://github.com/cuneytakinlar/pupil-ellipse-fit-error) 将该 Kaggle 页面列为 LPW 的 GT segmentation maps 下载处；本项目只使用数据，不复用该项目的代码、模型或椭圆拟合误差方法。
- 本地位置：`data/LPW/gt/imgs/` 与 `data/LPW/gt/masks/`。二者平行存放，不移动、不拆散。
- 文件命名：`<受试者>_<视频>_<帧号>.png`。例如 `01_01_0001.png`；`imgs` 与 `masks` 使用完全相同的文件名配对。
- 本地核验（2026-07-16）：图片 3,270 张、掩码 3,270 张，文件名双向差集均为 0；覆盖全部 66 段 LPW 视频。每段通常每隔 40 帧采样 50 张；`13_09`、`16_13`、`19_06` 分别只有 29、44、47 张。该资源属于**抽样掩码集**，未覆盖 130,856 帧的完整逐帧分割标注。
- 许可边界：该掩码为后续发布的第三方标注资源，不应表述为 LPW 原始作者提供的像素级真值。Kaggle 页面许可信息在使用前需人工留存；在确认前，按 LPW 的非商业科研限制使用，并保留 LPW 原始论文与掩码发布页链接。

## HMEPS 人眼辅助数据

- 名称：Human and Mouse Eyes for Pupil Semantic Segmentation（HMEPS）。
- 官方页：[Zenodo — HMEPS](https://zenodo.org/records/4488164)，DOI：10.5281/zenodo.4488164；许可证为 CC BY 4.0。
- 本地位置：`data/NN_human_mouse_eyes/fullFrames/` 为原图，`annotation/png/` 为掩码，`annotation/annotations.csv` 为筛选与元数据。
- 本地核验（2026-07-16）：`fullFrames` 当前仅有 4,285 张人眼 JPG，均为 128×128，且每张在 `annotation/png` 中有同名 PNG 掩码。`annotation/png` 共含 11,906 张掩码，其中还保留了未下载原图的鼠眼/其他实验记录，不能直接全量遍历。
- 训练筛选：只使用 `annotations.csv` 中 `exp=h`、`eye=1`、`blink=0` 的行，共 3,528 张。`exp` 字段实际为小写 `h`；排除鼠眼、无眼和眨眼帧。
- 掩码读取：HMEPS 的瞳孔标签存于掩码 PNG 的红色通道，加载时以该通道转换为二值掩码。

## 阶段 1 使用约定

1. 分割训练、验证和测试直接读取 `gt/imgs` 与 `gt/masks`，避免从 AVI 重拆帧造成错位。
2. 原始 `.txt` 中心点仅用于核验掩码导出的中心，以及补充中心定位评估；帧号应与 PNG 文件名对齐。
3. 切分必须以受试者为单位，禁止随机打散单帧；建议固定 16/3/3 个受试者作为训练/验证/测试，并将名单写入切分文件。
4. 开训前必须完成：掩码二值取值检查、图片/掩码尺寸一致性检查、同名配对检查，以及抽样叠加图人工检查。
5. HMEPS 只作为训练辅助数据；应先完成 LPW-only 基线，再与“LPW 训练集 + HMEPS 人眼训练集”对照。验证和最终测试均只使用按受试者隔离的 LPW。
6. 原始文件不离线改写。加载器统一将图像转为单通道、掩码转为 `0/1`，再对图像与掩码同步执行等比例缩放与补边；输入分辨率作为版本化配置确定，禁止直接拉伸以免改变瞳孔椭圆形状。

## 统一处理产物

- 生成程序：`scripts/prepare_pupil_segmentation_data.py`；依赖 Pillow，运行 `python scripts/prepare_pupil_segmentation_data.py --size 128`。
- 输出位置：`data/processed/pupil_segmentation_128/`。目录包含 `images/`、`masks/`、`manifests/samples.csv`、`dataset_summary.json` 与两张人工抽检预览图；所有均为生成数据，不进入 Git。
- 处理规则：原图统一为 8-bit 单通道 PNG；掩码统一为 8-bit 单通道 PNG，值为 0（背景）或 255（瞳孔）。LPW 掩码直接二值化；HMEPS 掩码取红通道后二值化。图像与掩码同步等比例缩放、居中补边到 128×128，图像采用双线性插值、掩码采用最近邻插值。
- 本次产物（2026-07-16）：LPW 3,270 对，HMEPS 人眼 3,528 对，合计 6,798 对；清单保留来源、受试者、序列、帧号、原始路径、缩放比例和补边参数。
- 空掩码：保留 21 对原始即为空的标注（LPW 20 对、HMEPS 1 对），未对其补画或删除。后续训练应明确决定它们作为有效负样本还是单独排除，并将该决定写入训练配置。
- 分辨率边界：128×128 用于阶段 1 的离线 FP32 基线。若上板实验需要 64×64，必须从原始数据重新运行本程序的 `--size 64`，不能对 128×128 产物再次缩放。
